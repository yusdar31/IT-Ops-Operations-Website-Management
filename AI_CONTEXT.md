**Tujuan Dokumen**: Memberikan konteks lengkap kepada AI Agent mengenai status terkini, arsitektur, dan langkah selanjutnya dari proyek migrasi **IT Operations Hub**.

---

## 1. Project Overview
Proyek ini adalah migrasi dari aplikasi monolith AWS IT Inventory System (berbasis PHP/Apache) menjadi aplikasi berarsitektur **Microservices** yang di-deploy ke **Kubernetes (K3s)** di atas AWS EC2.

## 2. Tech Stack & Arsitektur
- **Frontend**: React.js (Vite), Tailwind CSS, Framer Motion, 21st.dev / Shadcn UI primitives.
- **Backend APIs**: Node.js / Express.js.
- **Database**: MySQL 8.0 (Relational data) & Redis (Session, Pub/Sub, Queue).
- **Worker**: Node.js worker memproses background task dari Redis queue (mengirim email via AWS SES).
- **Infrastructure**: Docker Compose (Local Dev) -> ditargetkan ke Kubernetes (K3s) & Terraform.

### Daftar Services:
1. `services/auth-api` (Port 3001): JWT Authentication & RBAC (Role: admin, technician, viewer).
2. `services/asset-api` (Port 3002): CRUD Data Aset (Server, Network, dll).
3. `services/ticket-api` (Port 3003): Manajemen Tiket IT Helpdesk & Comments.
4. `services/notification-worker`: Consumer Redis antrian notifikasi, memanggil AWS SES.
5. `services/frontend` (Port 5173): Web Interface premium (Dark mode, Glassmorphism).

## 4. Catatan Penting untuk AI Agent
1. **Marker Antigravity**: Sebagian besar file kode yang telah di-generate sebelumnya ditandai dengan komentar `// Generated by Antigravity`, `<!-- Generated by Antigravity -->`, atau `/* Generated by Antigravity */`. Marker ini bisa dibiarkan atau dihapus jika dirasa mengganggu.
2. **Frontend Mock Login**: Pada `services/frontend/src/pages/Login.jsx`, terdapat logika bypass login (baris 25) agar web dapat diakses meskipun container backend API tidak berjalan. Logika *fallback* ini wajib dihapus saat di-deploy ke production.
3. **Working Directory**: Harap pastikan *Cwd (Current Working Directory)* selalu diarahkan ke struktur yang tepat (contoh: `services/frontend` atau root repository).
4. **Environment Variables**: Local testing membutuhkan `.env` di masing-masing service atau dijalankan sepenuhnya lewat `docker-compose.yml`. Di environment K8s nanti, kita akan menggunakan K8s Secrets.
